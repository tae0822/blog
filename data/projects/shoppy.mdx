##  2. Architecture & Key Features

### 🔐 Role-Based Access Control (Admin System)
I implemented a secure admin verification system using **Firebase Auth** and **React Context API**.

- **The Logic**: The app compares the logged-in user's UID with a pre-registered `admins` list in the Firebase Database.
- **AuthContext**: Only users with the `isAdmin` flag can access the "New Product" page. This prevents unauthorized users from even seeing or triggering the product upload UI.

### ☁️ Media Management with Cloudinary
- **Performance Benefit**: Offloaded heavy image delivery to Cloudinary’s CDN, significantly reducing load on Firebase and improving page load speed.




- **Workflow**: 
  1. Admin selects a file.
  2. The file is sent directly to Cloudinary's signed upload API.
  3. The returned **secure URL** is then saved to Firebase.
- **Result**: Fast image delivery via Cloudinary's Global CDN.

### 🛒 Real-time Cart with TanStack Query
- **State Synchronization**: Used `useMutation` with `invalidateQueries` to ensure **eventual consistency** between client UI and Firebase data.
- **User Experience**: Optimistic UI updates provide instant feedback before server confirmation.

## 🔥 4. Engineering Challenges & Solutions

### Securing Protected Routes & Persistent Auth
- **Challenge**: Preventing unauthorized users from accessing admin routes during authentication state hydration (e.g., page refresh).
- **Solution**: Used Firebase’s `onAuthStateChanged` observer inside a global AuthContext to delay rendering until the user's role is verified, eliminating UI flickering and access leaks.
